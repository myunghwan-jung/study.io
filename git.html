Git Usage Guide

1. Getting Started with Git

Run PowerShell in your working folder:

git config --global user.email "your_email"
git config --global user.name "your_name"

2. Using Git in a Code Editor

Initialize Git in the working folder:

git init

3. Staging

Stage a specific file:

git add filename

4. Committing Changes

Commit changes with a message:

git commit -m "description"

5. Other Commands

  • Stage multiple files at once:
    • git add file1 file2 file3
  • Stage all files:
    • git add .
  • View status:
    • git status
  • View commit history:
    • git log --all --oneline
  • Use --graph for visualization.

6. Built-in Git Integration in VSCode

VSCode has built-in Git integration, eliminating the need for terminal commands.

7. Viewing Differences Between Code Versions

  • Compare current and previous versions:
    • git diff
  • Use a diff tool:
    • git difftool
  • Compare with a specific commit:
    • git difftool commit_id
  • Compare two commits:
    • git difftool commit_id1 commit_id2

8. Branching

  • Create a new branch:
    • git branch branch_name
  • Switch to a branch:
    • git switch branch_name
  • Switch to the master branch:
    • git switch master

9. Branch Merging

Merge a branch into the current branch:

git merge branch_name

10. Conflict Resolution

Resolve conflicts manually when multiple branches modify the same file.

11. GitFlow

GitFlow introduces structured branch management:

  • Feature Development: Use feature branches for new features.
  • Release Process: Test changes in a release branch.
  • Hotfix: Fix critical issues in a hotfix branch.

12. Trunk-Based Development

Maintain only the main branch, creating feature branches as needed.

13. Merge Strategies

  • Use 3-Way Merge for the default merging strategy.
  • Use Fast-Forward Merge for a linear history:
    • git merge --no-ff branch_name

14. Rebase

Rebase and perform a fast-forward merge:

git rebase master
git switch master
git merge new_branch

15. Squash and Merge

Combine commits into a single commit:

git merge --squash branch_name
